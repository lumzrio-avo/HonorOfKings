# 王者荣耀信息管理系统 | Java OOP 项目

## 项目简介
独立开发的 Java 控制台游戏后台管理系统，涵盖英雄、装备、战队、比赛等核心实体，实现多对多双向对象关系绑定与完整业务功能闭环。

## 个人贡献声明
本项目为个人独立开发，负责全部系统架构设计、代码实现与文档输出。

- 系统架构：独立设计多层类体系，基于抽象类、继承、关联、聚合完成系统建模，设计 Person 抽象父类及玩家、管理员派生类
- 设计模式：运用策略模式实现装备使用率排行、胜率排行等不同排序逻辑，运用外观模式统一子系统调用接口
- 权限体系：搭建分级权限认证体系（管理员全数据增删改查 vs 玩家仅管理个人信息）
- 数据初始化：编写工具类自动构建样本数据并完成对象关联
- 文档输出：输出完整 UML 类图与模块设计文档

## 技术栈
Java（OOP、集合、流式编程）、策略模式、外观模式、UML

## 运行方式
1. 确保已安装 JDK（开发环境：Eclipse / IntelliJ IDEA）
2. 导入项目源码
3. 运行主类 Main.java
4. 按照控制台菜单提示操作

## 文件结构
- Person.java          # 抽象父类
- Player.java          # 玩家派生类
- Admin.java           # 管理员派生类
- Hero.java            # 英雄实体类
- Equipment.java       # 装备实体类
- Team.java            # 战队实体类
- Match.java           # 比赛实体类
- DataInit.java        # 数据初始化工具类
- AuthManager.java     # 权限认证管理类
- MatchManager.java    # 赛事管理类
- TeamManager.java     # 战队管理类
- Main.java            # 主入口 + 控制台交互界面
